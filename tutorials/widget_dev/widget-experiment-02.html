Setting the matplotlib plotting backendÂ¶

Matplotlib in Jupyter offers different outputs depending on the desired behavior. A figure may be saved to a file, or placed into its own window (like the ipython default), or embedded directly into the notebook alongside the code and explanatory material. This latter, popular, inline method can be achieved by executing the following line before figures:

%matplotlib inline

Additional, interactive functionality can be achieved with the notebook functionality:

%matplotlib notebook

Some common pitfalls in Jupyter widgets appear to center on the choice of backend, as evinced by this Stack Overflow question. This long GitHub Issue suggests that the widget-Jupyter interaction behavior appears to have changed in different versions of Jupyter and/or ipywidgets. After some experimentation, I found that the %matplotlib notebook backend seems to work:

%matplotlib --list

Compute the lightcurveÂ¶

We can assemble a photometric mask that includes all pixels except for the always-nan pixel.

In [9]:

nan_mask = np.all(np.isnan(tpf.flux), axis=0)
empty_mask = tpf.pipeline_mask * False
aper_mask = empty_mask
aper_mask[1:-1, 1:-1] = True
back_mask = ~aper_mask & ~nan_mask

In [11]:

tpf.plot(aperture_mask=aper_mask);
tpf.plot(aperture_mask=back_mask);

The outer annulus constitutes the background mask.

In [12]:

lc_aper = tpf.to_lightcurve(aperture_mask=aper_mask)/aper_mask.sum()
lc_back = tpf.to_lightcurve(aperture_mask=back_mask)/back_mask.sum()

Weâll flag outliers as any cadences above \(5\sigma\) in either the aperture or background, or possessing a nonzero quality flag.

In [13]:

_, outliers1 = lc_aper.remove_outliers(return_mask=True)
_, outliers2 = lc_back.remove_outliers(return_mask=True)

_, outliers3 = lc_aper.flatten().remove_outliers(return_mask=True)

outliers = outliers1 | outliers2 | (tpf.quality > 0.0) | outliers3

Perform the background subtraction.

In [16]:

lc = lc_aper - lc_back.flux
lc = lc[~outliers]

We will also want to drop all the outlier cadences in the tpf. lightkurve does not presently support the same fancy boolean indexing as the lightcurve class.

#Boolean indexing not supported-- this line of code will fail!
tpf[~outliers]

Instead we will have to manually mask each feature of interest.

def quick_look(lc=None):
    """
    Interact with a linked target pixel file and lightcurve

    Parameters
    ----------


    Returns
    -------
    ax : matplotlib.axes._subplots.AxesSubplot
        The matplotlib axes object.
    """
    try:
        from ipywidgets import interact
        from bokeh.io import push_notebook, show, output_notebook
        from bokeh.plotting import figure
        from bokeh.models import Span
        from bokeh.layouts import row
        output_notebook()
    except ImportError:
        raise ImportError('The quicklook tool requires Bokeh and ipywidgets.  See the Installation Guide.')
        return

    if lc is None:
        lc = tpf.to_lightcurve()

    title = "Quicklook lightcurve for {} target {}".format(tpf.mission, tpf.keplerid)
    p = figure(title=title, plot_height=300, plot_width=600)
    p.yaxis.axis_label = 'Normalized Flux'
    p.xaxis.axis_label = 'Time - 2454833 (days)'

    r = p.step(lc.time, lc.flux, color="#2222aa", line_width=3, )
    #r2 = p.line([lc.time[800], lc.time[800] ],[80, 115], color="#aa2222", line_width=3)
    vert = Span(location=800, dimension='height', line_color='firebrick', line_width=4)
    p.add_layout(vert)

    s2 = figure(plot_width=300, plot_height=300, title='Target Pixel File')
    s2.yaxis.axis_label = 'Pixel Row Number'
    s2.xaxis.axis_label = 'Pixel Column Number'

    s2_dat = s2.image([tpf.flux[0,:,:]], x=tpf.column, y=tpf.row,
                      dw=tpf.shape[2], dh=tpf.shape[1], dilate=True)#, palette="Spectral11")



    def update(f):
        #r2.data_source.data['x'] = [lc.time[f], lc.time[f]]
        vert.update(location=tpf.time[f])
        s2_dat.data_source.data['image'] = [tpf.flux[f,:,:]]
        push_notebook()

    show(row(p, s2), notebook_handle=True)
    interact(update, f=(0, 3550));
